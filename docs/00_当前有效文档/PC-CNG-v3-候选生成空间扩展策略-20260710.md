# PC-CNG v3 候选生成空间扩展策略

## 背景

当前瓶颈不在 loss，而在候选生成空间。已有 v3 edit decoder 能区分 observed / known-positive / hard-negative 角色，但候选池里高质量未观测 type-1 boundary negatives 不够多。

因此新增四类候选动作：

```text
heteroatom
regio
tautomer
low_yield_seed
```

目标是扩大候选空间，同时避免回到旧 MVP 的随机字符串扰动。

## 1. Heteroatom action

化学含义：

将新形成键的转移片段迁移到附近 heteroatom anchor，例如 N/O/S/P/halogen。

目的：

- 构造 chemoselectivity alternative；
- 捕捉同一反应上下文下的 competing nucleophile / electrophile site。

筛选：

- candidate anchor 必须是 heteroatom；
- product similarity 中等偏高；
- atom balance 较高；
- 不在 known-positive set。

## 2. Regio action

化学含义：

将新形成键迁移到同一 scaffold 中的近邻 anchor，构造 regioisomeric product。

目的：

- 对齐 RegioSQM20 / Science Advances 中最有价值的 type-1 negative；
- 学习 regioselectivity boundary。

筛选：

- anchor 距离真实反应中心较近；
- product similarity 较高；
- 不允许 exact known-positive；
- 优先同原子类型或同环局部环境。

## 3. Tautomer action

化学含义：

对 product 枚举 tautomer，构造 tautomeric alternative product。

目的：

- 捕捉产物互变异构造成的边界不确定性；
- 避免模型把可互变产物误判成任意错误产物。

实现：

- RDKit `TautomerEnumerator`
- 排除原始 product；
- 排除 known-positive；
- 保留相似度足够高的 tautomer。

## 4. Low-yield seed action

化学含义：

直接使用真实低产率 / failed / real_negative 反应作为 hard-negative seed。

目的：

- 引入真实实验失败信号；
- 避免所有负样本都来自规则生成；
- 对齐 Science Advances 中 HiTEA/HTE 真实失败数据使用方式。

注意：

- low-yield seed 可用于 BCE/feasibility classifier；
- 若用于 pairwise reward，需要后续构造相应 positive reference。

## 后续接入方式

建议下一轮训练矩阵：

1. real-only baseline
2. rule hard negatives from existing edit candidates
3. expanded hard-negative actions
4. expanded hard-negative actions + known-positive masked decoder
5. stacked ensemble

成功标准：

- 单模型 validation ROC-AUC/AUPRC 超过 real-only；
- 或 ensemble 中稳定贡献增益；
- 人工抽样时 heteroatom/regio/tautomer/low-yield seed 负样本明显比旧 MVP 更合理。
